Model Architecture

π0.5 — Vision-Language-Action

π0.5 is built on PaliGemma (Google's 3B VLM) with an added 300M action expert that communicates via cross-attention at every transformer layer. It generates actions through flow matching — a continuous diffusion process that denoises a chunk of Gaussian noise into a 1-second action sequence.

Flow Matching Loss

At each training step a random timestep t ∈ [0, 1] is sampled. The clean action is linearly interpolated with Gaussian noise to produce a noisy action xt. The model predicts the velocity field that maps noise back to the clean action:

xt = t · ε + (1 − t) · a
ut = ε − a
ℒ = MSE(vθ(xt, t, obs), ut)

At inference, 10 Euler denoising steps integrate the velocity field from t=1 to t=0, producing a 10-step action chunk.

Episode Phases

Phase 0a 300 steps

Shoulder joint (s0) sweeps from home position to align laterally with the cube while joints 1–6 stay folded.

Phase 0b 400 steps

Joints 1–6 interpolate from home to approach configuration at the fixed shoulder angle, descending toward the workspace.

Phase 1 ≤ 600 steps

Shoulder sweeps continuously in −Y, arm pushes the cube across the table. Episode terminates early on success.

Training Results

LoRA Fine-tuning

Full fine-tuning of π0.5 requires ~50 GB of GPU memory. Using LoRA (rank 16) on both PaliGemma and the action expert reduces peak usage to ~17 GB — fitting within a single RTX 5090 (32 GB). Only ~30–50M of 3B parameters are updated.

Key Findings

✅

Structured Motion

The policy produces task-relevant motion, not random noise. Shoulder and elbow dominate; wrist joints are correctly suppressed. Action magnitude ramps up monotonically — the model commits to the push trajectory with increasing confidence.

✅

Clean Convergence

Loss dropped 67.4% over 20k steps. Only 3 gradient clips in 198 logged steps, all in the first 1k. Param norm drifted just +0.4% — the pre-trained base model is essentially intact; only the LoRA adapters changed.

⚠️

Language Conditioning Collapse

Changing the prompt (e.g., "wave your hand") produced identical cube-push motion. Fine-tuning on a single task with one prompt removes language grounding — the visual scene alone drives the action. Multiple tasks with distinct outcomes would be required to restore prompt sensitivity.

✅

Efficient Fine-tuning

LoRA (rank 16) reduced GPU memory from ~50 GB (full FT) to ~17 GB, enabling training on a single RTX 5090. The async checkpoint OOM bug was resolved by switching to synchronous writes and closing background applications.
